# AGENTS.md

## Purpose
This directory provides shared utilities used across AthenaK: runtime helpers,
finite-difference operators, interpolation utilities, table/profile readers,
random-number generators, and small physics helpers.
See `../../AGENTS.md` for repository-wide conventions and workflow.

---

## Key Files and Responsibilities

### Runtime helpers
- `utils.hpp`: prototypes for `ShowConfig`, `ChangeRunDir`, and `CreateMPITag`.
- `show_config.cpp`: prints compile-time configuration (problem generator, precision,
  MPI/OpenMP flags).
- `change_rundir.cpp`: `mkdir` + `chdir` into the requested run directory; exits on
  failure to change directories.

### Finite-difference operators (auto-generated)
- `finite_diff.hpp`: templated, inline high-order operators generated by
  `py/write_FD.py` (do not edit by hand).
  - `Dx`: first derivative (scalar/vector/2D tensor overloads).
  - `Dxx`: second derivative (scalar/vector/2D tensor overloads).
  - `Dxy`: mixed second derivative (scalar/vector/2D tensor overloads).
  - `Lx`: upwinded first-derivative using sign of `vx` (scalar/vector/tensor overloads).
  - `Diss`: dissipation operator (vector-valued overload).
  - Supports `NGHOST = 2, 3, 4` with compile-time `if constexpr` stencils.

### Interpolation utilities
- `lagrange_interpolator.hpp` / `lagrange_interpolator.cpp`:
  - `LagrangeInterpolator` interpolates a single variable at a point using
    `2*ng` Lagrange weights per dimension (weights built on host).
  - Finds the owning `MeshBlock` for the point and records indices
    (`interp_indcs`) and weights (`interp_wghts`), then sums over the local
    stencil via `Kokkos::deep_copy` of scalar subviews.
  - `ResetPointAndInterpolate` updates the target coordinate and repeats setup.
  - `InterpolateTensor` is declared in the header but has no definition in the
    current source tree.

### Profile readers
- `profile_reader.hpp`:
  - `ProfileReaderHost` reads profile files on host, skipping the first line and
    parsing columns as `r`, `T`, `rho`, `v`.
  - `ProfileReader` copies host data into device `DvceArray1D` views and provides
    device-safe linear interpolation with endpoint clamping (`KOKKOS_INLINE_FUNCTION`).

### Table reader
- `tr_table.hpp` / `tr_table.cpp` (namespace `TableReader`):
  - `Table` loads a custom table format with header blocks (`<metadatabegin>`,
    `<scalarsbegin>`, `<pointsbegin>`, `<fieldsbegin>`, each terminated by
    `<...end>`), then reads a binary data payload.
  - Allocates one contiguous `double* data` buffer and builds `fields` pointers
    for coordinate arrays and field arrays.
  - Endianness is checked via metadata and optionally swapped using
    `tr_utils.hpp`.
- `tr_utils.hpp`: `SwapEndianness<T>` and `IsLittleEndian()` helpers.

### Random number utilities
- `random.hpp`:
  - `RNG_State` struct + `RanSt` (stateful uniform RNG) and `RanGaussianSt`.
  - `Ran2` and `RanGaussian` variants that use static internal state.
  - All RNG entry points are `KOKKOS_INLINE_FUNCTION` for device use.

### Small physics helpers
- `current.hpp`:
  - `pp` (parity of a permutation), `antisym` (4D antisymmetric symbol), and
    `get_detg_Fcon` (computes `det(g) * F^{mu nu}` from covariant 4-velocity and
    4-magnetic field components). All are `KOKKOS_INLINE_FUNCTION`.
- `sn_scheduler.hpp`:
  - `GetNthSNTime` returns the time of the nth SN event using FIRE-3 piecewise
    rates, converted from Myr into code time units (`KOKKOS_INLINE_FUNCTION`).

## Usage in the Codebase

- `finite_diff.hpp`: used by `z4c/` and `dyn_grmhd/` for spatial derivatives.
- `lagrange_interpolator.hpp`: used by `z4c/compact_object_tracker.cpp` to
  interpolate Z4c/ADM/MHD fields at compact-object positions.
- `profile_reader.hpp`: used by CGM cooling-flow problem generators in `src/pgen/`.
- `tr_table.hpp`: used by `pgen/dyngr_tov.cpp` and `eos/primitive-solver/eos_compose.cpp`.
- `random.hpp`: used by `srcterms/turb_driver.*` and `pgen/rt.cpp`.
- `current.hpp`: used by `outputs/derived_variables.cpp` for current diagnostics.
- `sn_scheduler.hpp`: used by `particles/particles.cpp` and
  `pgen/cgm_cooling_flow_amr_metals.cpp`.
- `ShowConfig` and `ChangeRunDir`: called from `src/main.cpp`.

---

## Cautions and Notes

- `finite_diff.hpp` is auto-generated; update `py/write_FD.py` instead of editing
  the header directly.
- `Table::ReadTable` reads the binary payload as `double` and includes a
  `FIXME` for single-precision support.
- `CreateMPITag` is declared in `utils.hpp`, but no definition exists in
  `src/utils` or elsewhere in the current tree.
- `ProfileReaderHost` throws `std::runtime_error` on file open failure or if no
  valid data lines are parsed.
