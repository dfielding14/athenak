# Example: Binary Neutron Star Merger

Fully dynamical GRMHD evolution of an SGRID binary neutron-star data set.

## Problem Generator
- **SGRID BNS importer**: `src/pgen/sgrid_bns.cpp`

The problem generator interfaces with the external **libsgrid** library to read
initial data and populate the ADM/Z4c/MHD state vectors.

## Required Assets

1. **libsgrid static library**
   Place the library produced by SGRID (or the Elliptica toolkit) at
   `sgrid/lib/libsgrid.a`. CMake links this archive when the binary merger input
   deck is used.

2. **Initial data directory**
   The shipped input deck expects the dataset
   `sgrid_26_26_26/` to live in the run directory (relative to the executable).
   Fetch this directory from your SGRID/Elliptica output and copy it into the
   AthenaK run directory before launching the simulation.

> ⚠️ These assets are not versioned in the repository. The run will abort if the
> library or dataset is missing.

## Input File

```
inputs/dyngr/sgrid_bns_amr.athinput
```

Key blocks:
- `<mesh>`/`<meshblock>`: 192³ domain with 32³ MeshBlocks.
- `<mesh_refinement>`: 7 AMR levels with tracker-driven refinement.
- `<mhd>`: Piecewise-polytropic EOS (piecewise_poly) with FOFC disabled.
- `<z4c>`/`<z4c_amr>`/`<adm>`: Gauge and wave-extraction parameters.
- `<problem>`: Points to `datadir = sgrid_26_26_26`, sets binary orbital
  parameters, and enables custom history/refinement callbacks.

## Build & Run

```bash
# Configure once (built-in problem set already includes sgrid_bns)
cmake -S . -B build
cmake --build build -j $(sysctl -n hw.ncpu 2>/dev/null || nproc)

# Copy the libsgrid library and initial data into place, then run
cp /path/to/libsgrid.a sgrid/lib/
cp -r /path/to/sgrid_26_26_26 .
./build/src/athena -i inputs/dyngr/sgrid_bns_amr.athinput
```

The problem generator creates a working directory named `SGRID/` to stage
temporary files and histories generated by libsgrid. Simulations are resource
intensive (large memory footprint and long runtimes); inspect the mesh size and
AMR parameters before launching on production hardware.

## Outputs

The input deck writes:
- History diagnostics (`bns.hst`)
- Binary mesh dumps for primitive/conserved variables (`bns.block*.out*.bin`)
- ADM/Z4c metric fields (`bns.block*.out*.bin`)
- Restart checkpoints (`bns.#####.rst`)

Wave extraction radii are set in `<z4c>` (`extraction_radius_[1-3]`) and
waveforms are written to the history stream.
