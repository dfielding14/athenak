## Figure Provenance

The images are generated by `vis/python/plot_projection_examples.py` from evolved
`projection_layout = native_amr` patch files written by these shipped example inputs.
The reader in `vis/python/projection.py` composes patch moments into display grids. The
AMR statistics figure uses a blast configured for four adaptive levels above the root
grid and composes at the finest level present; its third panel maps the finest native
leaf-patch level contributing to every display bin. The evolved slab product used for
the checked-in image contains 776 patch records over leaf levels 1 through 4 and
composes to a `384 x 384` display image at level 4.

- `inputs/hydro/projection/blast_column_density.athinput`
- `inputs/hydro/projection/blast_amr_weighted.athinput`
- `inputs/mhd/projection/field_loop3d_current.athinput`

The hydro examples require a build configured with `-D PROBLEM=blast`; the MHD example
requires `-D PROBLEM=field_loop`. The AMR statistics figure uses the slab
`0 <= x1 <= 0.1`, and the profile figure reduces over bounded `x1,x2` intervals to
demonstrate a 3D-to-1D output. The module page documents
`single_file_per_node=true` as the production native-AMR scaling path and treats
explicit guarded `projection_layout = uniform` output as a small reference mode only.

## Local Pages Validation

Copy both candidate module pages and four static images into a detached `origin/gh-pages`
worktree, register the navigation entries above, then run:

```sh
cd docs
make clean html SPHINXOPTS="-W --keep-going"
```

This validates MyST syntax, image inclusion, and navigation integration against the
documentation branch used for publication.

## Related Output-Sharding Dependency

This branch merges the general output-sharding infrastructure documented by
`docs/source/modules/output_file_sharding.md`. When publishing both feature branches,
retain that page as the sharding reference and publish
`docs/source/modules/projection_outputs.md` for projection moments, composition, and
scaling guidance.
